## 참조의 지역성과 작업 집합



### 프로그램의 실행 특성

&nbsp;&nbsp;현대의 대부분 운영체제는 요구 페이징을 사용한다. 필요할 때 페이지를 적재하는 요구 페이징은 근본적으로 페이지 폴트를 동반하고 이로 인해 디스크 입출력이 발생하는데 그럼에도 불구하고 사용하는 이유는 다음 2가지 프로그램 실행 특성 때문이다.

- [ ] 참조의 지역성
  - 적재된 페이지를 한 동안 액세스
  - 시간 지역성
  - 공간 지역성
- [ ] 작업 집합



### 작업 집합

- [ ] 작업 집합
  - **일정 시간 범위** 내에 프로세스가 액세스한 페이지들의 집합, 즉 현재 프로세스의 실행에 필요한 페이지들의 집합
  - 참조의 지역성 때문에 작업 집합은 뚜렷이 형성됨



&nbsp;&nbsp;작업 집합에 포함된 페이지의 개수는 그리 크지 않다. 이들이 모두 메모리에 적재되어 있다면 프로세스는 한 동안 페이지 폴트 없이 최고의 실행 성능을 얻을 수 있다. 그러므로 운영체제는 프로세스에게 작업 집합을 적재할 수 있는 정도의 메모리를 할당하고, 작업 집합에 포함된 페이지가 스왑-아웃되지 않도록 관리할 필요가 있다.  

&nbsp;&nbsp;프로세스의 실행 중 갑자기 페이지 폴트가 계속된다면 작업 집합 페이지를 메모리에 적재하고 있는 과정이라고 생각해도 된다. 시간이 지나면 페이지 폴트가 줄고 작업 집합은 일정 시간의 구간에서 뚜렷이 형성된다.  



![screenshot-8192846](/assets/images/posts_img/2024-06-07-chap10-virtual-memory/screenshot-8192846.png)

  

&nbsp;&nbsp;3번의 페이지 폴트로 페이지 2,100, 5, 20이 메모리에 적재되었고, main() 함수는 for() 문이 끝날 때까지 이 4개의 페이지들을 참조하면서 **한 동안 실행**된다. 이들이 바로 main() 함수가 실행되는 동안의 **작업 집합**이다.  

  

### 작업 집합 이동

&nbsp;&nbsp;참조의 지역성이 다른 메모리 영역에 액세스 시 다시 페이지 폴트가 증가하면서 새로운 작업 집합이 형성되기 시작할 것이다.

### 스래싱 발생 관측과 예방

- [ ] 스래싱
  - **작업 집합**에 포함된 페이지들이 충분히 메모리에 올라와 있지 않는 경우에 발생
  - 작업 집합 페이지들을 수용할 충분한 메모리를 할당하는 알고리즘을 통해 스래싱 예방





### 요구 페이징의 필수 알고리즘

&nbsp;&nbsp;요구 페이징의 성능은 작업 집합에 속한 페이지들을 메모리에 적재한 상태로 프로세스를 실행시킬 수 있느냐에 따라 달려있다. 이를 위해 2개의 알고리즘이 필요하다.  

- [ ] 프레임 할당
- [ ] 페이지 교체

  

## 프레임 할당

​    

- [ ] 정의
  - 프로세스에게 할당할 프레임의 개수를 결정

- [ ] 목표
  - 프로세스의 작업 집합에 포함된 페이지들을 충분히 수용할만한 개수의 프레임을 할당하여 페이지 폴트를 줄이는 것

- [ ] 방법

  - 균등 할당

    - 프로세스 크기와 관계없이 모든 프로세스에게 동일한 개수의 프레임을 할당

    - 작은 프로세스는 필요 이상의 많은 프레임을 할당 받고 큰 프로세스는 작업 집합을 담기에 프레임이 부족하여 페이지 폴트가 자주 발생할 수 있는 허점 존재

  - 비례 할당

    - 프로세스 크기에 비례하여 프레임을 할당

    - 프로세스 크기를 명확히 알기 어려움

- [ ] 프로세스에게 할당할 적정 프레임 수
  - 작업 집합을 약간 넘나드는 수준이 적합

##  페이지 교체

  

- [ ] 정의
  - 요청된 페이지가 프레임에 없고, 페이지를 적재할 빈 프레임도 없는 경우, 희생 프레임을 선택하여 요청된 페이지를 적재하는 과정
    - 희생 페이지가 수정된 페이지(M bit=1)라면 스왑-아웃

- [ ] 목표
  - 현재 작업 집합에 포함되지 않거나 가까운 미래에 참조되지 않을 페이지를 희생 페이지로 선택하여 페이지 폴트 횟수를 줄이는 것

  

> &nbsp;&nbsp;메모리 프레임에 있는 페이지들 중, 현재 작업 집합에 속하지 않거나 가까운 미래에 참조되지 않을 페이지를 어떻게 판단할 수 있을까?

  

- [ ] 희생 프레임의 선택 범위
  - 지역 교체
    - 페이지 적재를 요청한 프로세스에게 할당된 프레임들 중 희생 프레임을 선택
      - 다른 프로세스의 프레임을 건들지 않기 때문에 스래싱이 전파되지 않음
  - 전역 교체
    - 전체 프레임 중에서 희생 프레임을 선택
    - 일반적으로 전역 교체가 지역 교체보다 페이지 폴트를 덜 유발시킴

  

- [ ] 페이지 교체(희생 페이지 선택) 알고리즘 종류
  - 최적 교체(Optimal Page Replacement, OPT)
    - 가까운 미래에 사용될 가능성이 가장 낮은 페이지 선택
    - 실현 불가능
  - FIFO(First In First Out)
    - 가장 오래된 페이지 선택
  - LRU(Least Recently Used)
    - 가장 오래전에 참조된 페이지 선택
  - Clock
    - 프레임 당 **참조 비트** 사용
      - CPU가 페이지를 참조할 때마다 참조 비트를 **1**로 수정
    - 프레임들을 원형 큐로 관리
      - 검색을 시작하는 원형 큐의 위치를 **포인터**라고 함
      - 원형 큐를 따라 현재 포인터에서 참조 비트가 **0**인 프레임을 만날 때까지 시계 방향으로 이동
        - 이때, 참조 비트가 **1**이면 **0**으로 바꾸면서 진행
